## Import a GDTF fixture

<Steps>
<Step>
### Open the import dialog

Choose **File ▸ Import GDTF Fixture…**. A native file dialog opens, filtered to `.gdtf` files.
</Step>
<Step>
### Select a `.gdtf` file

Pick a file from disk and confirm. Glowstone parses the ZIP, reads `description.xml`, and extracts the modes, channels, wheels, geometry, and optics.
</Step>
<Step>
### The fixture lands in the scene

The parsed profile is added to the project [Library](/docs/library) and a new instance is placed in the scene at position `(0, 4, 0)`. The new fixture is selected and ready to configure in the Inspector.
</Step>
</Steps>

<Callout type="idea" title="Drag and drop">
You can also drag a `.gdtf` file straight onto the Glowstone window instead of using the menu. The result is identical: parsed, added to the Library, and placed in the scene.
</Callout>

Imported fixtures are tagged with a light-blue **Imported** provenance chip so you can tell them apart from built-in fixtures and those downloaded from [GDTF Share](/docs/library/gdtf-share).

## What's inside a `.gdtf` archive

A `.gdtf` file is a renamed ZIP. Glowstone reads its contents directly.

Models are read from `models/gltf_high/` when present, otherwise `models/gltf/`; wheel and thumbnail media are `.png`.

## What gets extracted

When a GDTF is parsed, Glowstone reads and retains the complete fixture definition.

These optical values come straight from the fixture's `description.xml` and are shown in the [Profile Editor](/docs/library/profile-editor).

<Callout type="info" title="Geometry hierarchy">
The `Axis` nodes for pan and tilt rotate the head, while the `Beam` node carries the optics. Glowstone converts GDTF's +Z-up coordinates to the app's +Y-up space during import.
</Callout>

## Fixtures travel with your project

Once imported, the original `.gdtf` archive bytes are bundled into the `.glow` project file. You do not need to ship the original `.gdtf` files alongside the project — on open, each fixture's profile is re-parsed from the bundled archive, so geometry, wheels, and thumbnails come back intact.

<Callout type="success">
Open the same `.glow` project on another machine and every imported fixture loads with its real model and wheels intact, no external files required.
</Callout>

## FAQ

<Accordions type="single">
  <Accordion title="Where does an imported fixture appear?">
    It is added to the project Library and immediately placed in the scene at `(0, 4, 0)`, selected and ready to edit.
  </Accordion>
  <Accordion title="Can I change the DMX mode after import?">
    Yes. Open the fixture's profile and pick a mode in the DMX Modes tab. Switching modes reconfigures the available channels without removing the fixture. See <a href="/docs/library/profile-editor">DMX Modes &amp; Channels</a>.
  </Accordion>
  <Accordion title="What if a slot has no gobo image?">
    Colour wheel slots without media display a colour swatch derived from the GDTF CIE xyY colour data instead of an image.
  </Accordion>
  <Accordion title="Do I need the .gdtf file later?">
    No. The archive is bundled into the `.glow` project, so the original `.gdtf` is only needed at import time.
  </Accordion>
</Accordions>
