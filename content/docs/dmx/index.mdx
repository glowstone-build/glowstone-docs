## How the signal flows

<Steps>
<Step>
### Console sends DMX
Your console outputs DMX over **Art-Net** (UDP `6454`) or **sACN / E1.31** (UDP `5568`) onto the network.
</Step>
<Step>
### Connectivity receives and merges
A background receiver binds the sockets, tracks each live source and merges overlapping universes per the chosen merge policy. See [Connectivity](/docs/dmx/connectivity).
</Step>
<Step>
### The patch maps fixtures
The patch table assigns every fixture a **universe**, a start **address** and a channel **footprint**. See [Patching Fixtures](/docs/dmx/patching).
</Step>
<Step>
### Decode drives the optics
Once per frame, the current universe snapshot is applied to each enabled fixture and DMX values become degrees, levels and colours.
</Step>
</Steps>

<Callout type="info" title="Universes versus addresses">
A fixture is located by **universe + address**. A 65-fixture rig with an 8-channel footprint fills universe `1` (512 channels) and spills the last fixture into universe `2`. Auto-patch wraps to the next universe automatically.
</Callout>

## Protocols at a glance

<Tabs items={['Art-Net', 'sACN']}>
  <Tab value="Art-Net">
  Broadcast on UDP `6454`. Glowstone accepts all Art-Net universes regardless of subscription. Because the protocol carries no per-source priority, Glowstone assigns Art-Net a single fixed priority (default `100`).
  </Tab>
  <Tab value="sACN">
  Multicast on UDP `5568` (ANSI E1.31). Glowstone joins one multicast group per universe in your subscription list, so sACN universes you don't list aren't received — unlike Art-Net, an empty list joins nothing. Each packet carries its own priority (`0`–`200`); preview-data and stream-terminated packets are ignored.
  </Tab>
</Tabs>

<Callout type="idea">
Both protocols can run at once. If they overlap, the merge policy decides which source wins each channel — lower the Art-Net priority to let sACN take precedence.
</Callout>

## Common questions

<Accordions type="single">

  <Accordion title="My fixtures aren't moving — where do I start?">
    Confirm the source is live in the [Connectivity](/docs/dmx/connectivity) Sources table, then check the fixture is **enabled** and patched to the universe and address your console is driving. The [DMX Monitor](/docs/dmx/dmx-monitor) grid shows which channels are arriving.
  </Accordion>
  <Accordion title="What happens when two sources send the same universe?">
    The merge policy resolves it. **Priority + HTP** (default) lets the highest-priority source win, with per-channel highest-takes-precedence among equal priorities. See [Connectivity](/docs/dmx/connectivity).
  </Accordion>
</Accordions>
