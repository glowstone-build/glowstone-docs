## How input works

A single background thread binds UDP sockets, receives packets, merges them per universe, and publishes the result to the renderer. The [patch table](/docs/dmx/patching) then maps each fixture to a universe and address, decoding the merged values every frame to drive fixture optics.

The receiver starts automatically on the first interactive frame, so in typical use you do not need to start anything by hand. Open the panel from the **Connectivity** tab at the bottom of the window. When running, the **Receive DMX** toggle shows a status dot with the bound protocols and source count (e.g. `Art-Net + sACN · 2 source(s)`); when stopped it reads `stopped`.

## Protocols

Glowstone speaks two industry protocols. Both can run at the same time, and their data merges according to your [merge policy](#merge-policy).

<Tabs items={['Art-Net', 'sACN / E1.31']}>
  <Tab value="Art-Net">
    - **Port**: UDP `6454`
    - **Transport**: broadcast — Glowstone accepts **all** Art-Net universes regardless of the universe list.
    - **Addressing**: Art-Net Port-Addresses map to **1-based** universe numbers (Art-Net port 0 → universe `1`).
    - **Priority**: the protocol carries no per-source priority, so Glowstone assigns every Art-Net source a fixed [Art-Net priority](#art-net-priority).
    - **Loss detection**: from the packet sequence number (`0`–`255`). A sequence of `0` disables loss counting.
  </Tab>
  <Tab value="sACN / E1.31">
    - **Port**: UDP `5568` (ANSI E1.31-2018)
    - **Transport**: multicast — Glowstone joins one group **per subscribed universe** (group `239.255.{hi}.{lo}`).
    - **Addressing**: natively 1-based (universe `1` = `239.255.0.1`).
    - **Priority**: each packet carries its own priority (`0`–`200`).
    - **Identity**: a source is tracked by its CID (a stable UUID), not its IP address.
    - Only the standard DMX start code (`0x00`) is accepted. Preview and stream-terminated flags are honoured.

    <Callout type="info">sACN is multicast, so Glowstone only receives a universe whose group it has explicitly joined. List every sACN universe you want — an empty list joins no groups (the socket still binds the port, but the OS delivers no multicast traffic).</Callout>
  </Tab>
</Tabs>

<Callout type="idea">Art-Net and sACN can coexist on the same NIC as the console, and multiple Glowstone instances can listen on one host. Sockets use `SO_REUSEADDR` / `SO_REUSEPORT` so nothing blocks the port.</Callout>

### Merge policy

When more than one source sends the same universe, the merge policy decides which values reach the renderer.

<TypeTable
  type={{
    'Priority + HTP': {
      description: 'Only the highest-priority source(s) contribute; among equal priorities, the highest value wins each channel (Highest Takes Precedence).',
      type: 'default',
    },
    'HTP': {
      description: 'Priority is ignored. The highest value across all sources wins each channel.',
      type: '',
    },
    'Latest': {
      description: 'The most recently updated source wins the entire universe. Older sources are discarded.',
      type: '',
    },

### Art-Net priority

Because Art-Net packets carry no priority, Glowstone stamps a single fixed value (`0`–`200`, default `100`) onto every Art-Net source. Lower it below a console's sACN priority to let sACN win a [Priority + HTP](#merge-policy) merge, or raise it to let Art-Net win.

### Reapply

Some settings re-bind sockets and so are **deferred** until you press **Reapply**, which stops and restarts the receiver. The button is only enabled while the receiver is running.

<Callout type="warn" title="Which changes need Reapply">Changing the **protocols** (Art-Net/sACN) or the **bind interface** requires pressing **Reapply** to take effect. The **universe list**, **merge policy**, and **Art-Net priority** all apply live.</Callout>

## Monitoring live sources

The **Sources** table lists every source currently sending packets, with one row per source.

<TypeTable
  type={{
    Proto: { description: 'Protocol: Art-Net or sACN.', type: '' },
    Source: { description: 'ip:port for Art-Net; for sACN the source name with the CID hex prefix in parentheses (just the CID prefix if no name was sent).', type: '' },
    Universes: { description: 'Sorted list of universes this source has sent, e.g. 1-3, 5.', type: '' },
    Prio: { description: 'Source priority — fixed for Art-Net, per-packet for sACN.', type: '0–200' },
    FPS: { description: 'Smoothed packets per second (exponential moving average).', type: 'Hz' },
    Lost: { description: 'Cumulative packet-loss count from sequence gaps (wrap-aware).', type: 'count' },
    Seen: { description: 'Seconds since the last packet from this source.', type: 's' },
  }}
/>

The **FPS** column is colour-tiered so you can read network health at a glance:

| FPS | Colour | Meaning |
| --- | --- | --- |
| ≥ 30 Hz | Green | Healthy |
| 10–30 Hz | Yellow | Marginal |
| Below 10 Hz | Red | Conflict / lossy |

<Callout type="info">A source idle for more than **2.5 s** is dropped from the merge and removed from the table.</Callout>

## Connect a console

The flow is the same for any console: configure its output, enable the matching protocol in Glowstone, confirm the source appears, then patch to matching addresses.

<Steps>
<Step>
### Output from the console
On the console, enable Art-Net or sACN output and point it at the universe and network interface you want Glowstone to receive. See the [console walkthroughs](#console-walkthroughs) below for grandMA3 and MagicQ.
</Step>
<Step>
### Match the protocol in Glowstone
In the Connectivity panel, tick **Art-Net** or **sACN** to match the console, leave **Bind interface** at `0.0.0.0`, and press **Reapply**.
</Step>
<Step>
### Subscribe the universe (sACN only)
For sACN, add the console's universe to the **sACN universes** list — sACN is multicast, so an unlisted universe is never joined. Art-Net needs no subscription.
</Step>
<Step>
### Confirm the source
Watch the **Sources** table — the console should appear with green FPS and its universe list. If nothing shows, see [Troubleshooting](#troubleshooting).
</Step>
<Step>
### Patch and play
Patch fixtures to matching addresses in the [patch table](/docs/dmx/patching) or the **Fixture Manager** — select fixtures and use **Patch seq** to assign them sequentially from a starting universe/address by footprint. As the console fades the rig, the 3D fixtures update live.
</Step>
</Steps>

<Callout type="success">Once a green source is listed and fixtures are patched, the [DMX Monitor](/docs/dmx/dmx-monitor) shows live channel levels and the viewport reacts in real time.</Callout>

### Console walkthroughs

<Tabs items={['grandMA3', 'MagicQ']}>
<Tab value="grandMA3">

<Steps>
<Step>
### Configure the console output
On the grandMA3, open **Setup > DMX > Output**. Add an **Art-Net** node or an **sACN** universe, set the **universe** number (start with `1`), and choose the **network interface / IP**.
</Step>
<Step>
### Enable the protocol in Glowstone
Open the **Connectivity** tab at the bottom of the window. Tick **Art-Net** or **sACN** to match the console's output.
</Step>
<Step>
### Set the bind interface
Leave **Bind interface** at `0.0.0.0` to listen on all NICs — the most reliable choice, especially for sACN multicast across multiple interfaces. Then click **Reapply** to re-bind the sockets.
</Step>
<Step>
### Subscribe the universe (sACN only)
If you chose sACN, add the universe to the **sACN universes** list (e.g. `1`, or a range like `1-4`). Each entry joins one sACN multicast group, so a universe missing from this list is never received. Art-Net ignores this field — it is broadcast and accepts every universe regardless. Universe-list and merge edits apply live; protocol and interface changes need **Reapply**.
</Step>
<Step>
### Confirm Receive DMX is on
Tick **Receive DMX** to bind the sockets. The status line beside it shows which protocols bound (e.g. `Art-Net + sACN`) and the live source count; when off it reads `stopped`.
</Step>
<Step>
### Verify the source appears
Check the **Sources** table. You should see a row with **Proto** = Art-Net, the console's `ip:port` under **Source**, the universes it is sending, and a green **FPS** reading.
</Step>
<Step>
### Patch fixtures to matching addresses
Patch your Glowstone fixtures to the same universes and addresses as the console's rig. Select the fixtures and press `P` to open the Patch dialog (prefilled with the next free universe and address), or use the Fixture Manager's bulk **Patch seq** / **Set U** tools. See [Patching](/docs/dmx/patching).
</Step>
<Step>
### Play
Fade the rig on the console. The patched fixtures in the 3D Viewport update in real time.
</Step>
</Steps>

</Tab>
<Tab value="MagicQ">

<Steps>
<Step>
### Configure the console output
In MagicQ, open **Cues > Outputs** (the DMX I/O view). Select **Art-Net** or **sACN** — sACN is preferred on modern networks — and set the **universe** and **IP** to suit your network. Start with universe `1`.
</Step>
<Step>
### Enable the protocol in Glowstone
Open the **Connectivity** tab and tick **sACN** (or **Art-Net**) to match MagicQ's output.
</Step>
<Step>
### Set the bind interface
Leave **Bind interface** at `0.0.0.0`, then click **Reapply** to re-bind. sACN binds to `0.0.0.0` regardless, for reliable multicast receive.
</Step>
<Step>
### Subscribe the universe (sACN)
Add the MagicQ universe to the **sACN universes** list (e.g. `1`). For sACN you must list every universe you want — each entry joins its multicast group, and a universe not in the list is never received. (Art-Net is broadcast and ignores this field.)
</Step>
<Step>
### Confirm Receive DMX is on
Ensure **Receive DMX** is ticked; the status line shows `sACN` (or `Art-Net + sACN`) bound and the source count.
</Step>
<Step>
### Verify the source appears
In the **Sources** table, look for a row with **Proto** = sACN and MagicQ's **CID** (a hex prefix, plus the source name if MagicQ sends one). A steady green **FPS** confirms a healthy stream.
</Step>
<Step>
### Patch fixtures to matching addresses
Patch fixtures to the universes and addresses MagicQ is driving. Select fixtures and press `P` to open the Patch dialog, or use the Fixture Manager's bulk patch tools. See [Patching](/docs/dmx/patching).
</Step>
<Step>
### Play
Run your cues on MagicQ and watch the rig respond in the Viewport.
</Step>
</Steps>

## Troubleshooting

<Accordions type="single">
  <Accordion title="No sources appear in the table">
    - Confirm the console's Art-Net/sACN output is enabled and aimed at the right universe and IP.
    - Check firewall rules for UDP `6454` (Art-Net) and `5568` (sACN).
    - If Glowstone and the console are on different NICs, set **Bind interface** to `0.0.0.0` and press **Reapply**.
    - For sACN, verify the console's universe is in your subscription list — sACN is multicast, so unlisted universes are never joined.
  </Accordion>
  <Accordion title="The Lost column keeps climbing">
    Packet loss comes from sequence gaps, usually network congestion. Check the **FPS** column: a value below 30 Hz suggests the console is throttling or the link is lossy. If two sources are fighting for the same universe, review the [merge policy](#merge-policy).
  </Accordion>
  <Accordion title="The wrong universe is coming through (or none)">
    - For sACN, the **sACN universes** list must include the console's universe; ranges like `5-8` expand automatically.
    - Art-Net accepts every universe regardless of the list — if it still seems wrong, confirm the console's Art-Net Port-Address maps to the 1-based universe you expect (Art-Net port 0 → universe `1`).
    - After editing protocols or the bind interface, press **Reapply**.
  </Accordion>
  <Accordion title="Sources are listed but fixtures do not move">
    The source is arriving but the [patch](/docs/dmx/patching) may not match. Verify each fixture is enabled and its universe/address line up with the console's rig, and confirm the GDTF mode footprint is correct.
  </Accordion>
</Accordions>
