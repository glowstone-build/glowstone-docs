The shaping wheels sit between the source and the lens, after the dimmer and before focus and zoom: animation wheel → gobo wheels → colour → prism → frost. Each wheel is a separate optical component declared by the fixture's [GDTF mode](/docs/library/dmx-modes), and each one runs its own motion independently. This page covers gobos, the animation wheel, prisms, and frost — colour wheels and CMY live on the [Colour](/docs/fixtures/color) page.

## How shaping wheels are controlled

Each wheel component is one `WheelControl` with up to four normalized `0..1` fields: `value`, `index`, `spin`, `shake`. The Inspector renders one block per component under **Optics → WHEELS**, generated from the active mode's GDTF chain.

The first row carries `value` but is labelled by the component name (e.g. `Gobo1 · <wheel name>`), with a tooltip of **select** on gobo/colour wheels and **insert** on prism / animation / frost. The other rows are labelled `index`, `spin`, and `shake`.

<TypeTable
  type={{
    value: {
      description: 'Slot selection (gobo) or insertion amount (prism / animation / frost). 0 = open or removed, 1 = fully selected / inserted.',
      type: '0–1',
      default: '0',
    },
    index: {
      description: 'Indexed rotation of the inserted element — the gobo image or the prism facet ring. 0–1 wraps one full turn.',
      type: '0–1',
      default: '0',
    },
    spin: {
      description: 'Continuous rotation/scroll speed (bipolar). 0.5 = stopped, below = CCW, above = CW.',
      type: '0–1 (0.5 centre)',
      default: '0.5',
    },
    shake: {
      description: 'Shake / oscillation amount (gobo & colour wheels only). The indexed element sways back and forth. 0 = no shake, 1 = maximum.',
      type: '0–1',
      default: '0',
    },
  }}
/>

<Callout type="info" title="Which controls appear">
Not every row shows on every wheel. `value` always shows. `index` shows when the mode declares indexed rotation (and always on a prism). `spin` shows when the mode declares continuous spin (and always on colour / animation / prism). `shake` shows on **gobo and colour wheels only**. Availability is read from the active DMX mode, so different modes of the same fixture can expose different controls.
</Callout>

## Gobo wheels

Selecting a slot rotates the disc from its open hole to the chosen pattern, projected as a cookie that shapes both the beam shaft and the floor pool at once.

<Callout type="idea">
The gobo disc is always physically present in the shaft, even on the open slot (a clear hole). Because of that, selecting a gobo animates the disc spinning into place rather than popping the pattern on instantly.
</Callout>

### Selecting and rotating a gobo

<Steps>
<Step>
### Pick a slot
Drag `value` from `0` (open) toward `1` to step through the wheel's slots. The wheel slews to the selected slot at about 8 slots/s, briefly showing the physical split and holder gap as it travels, then snaps to park on the nearest whole slot.
</Step>
<Step>
### Set an indexed angle
Use `index` to rotate the projected pattern to a fixed angle. `0–1` wraps one full turn. This rotates the gobo image only, independent of which slot is selected.
</Step>
<Step>
### Or spin it continuously
Set `spin` away from the `0.5` centre to rotate the pattern non-stop — below centre runs CCW, above runs CW. At full spin the image rotates at about `2.4` turns/s (≈876°/s).
</Step>
<Step>
### Add shake
Raise `shake` to oscillate the pattern back and forth. Frequency runs `1–9 Hz` depending on the amount, with roughly `20°` of sway at maximum.
</Step>
</Steps>

<Callout type="info">
Focus controls how crisp the cookie is, and frost softens its edge. See [Beam parameter reference](/docs/fixtures/parameters) for `focus` and how the in-focus point maps to a real-world distance.
</Callout>

### Multiple gobo wheels

A fixture can carry several gobo wheels (`Gobo1`, `Gobo2`, …), each with its own `WheelControl`. They are listed as separate blocks in the Inspector and render in chain order, so you can rotate one pattern over another.

## Animation wheel

A fixture has at most one active animation wheel, scrolling texture across the beam.

<TypeTable
  type={{
    value: {
      description: 'Insertion amount. Above ~0.01 the wheel is inserted into the beam.',
      type: '0–1',
      default: '0',
    },
    spin: {
      description: 'Continuous scroll. 0.5 = stopped, away from centre scrolls the texture; full spin reaches about 1.95 turns/s.',
      type: '0–1 (0.5 centre)',
      default: '0.5',
    },
  }}
/>

<Callout type="info">
Insert the animation wheel with `value`, then drive `spin` for movement. The two combine well with a gobo — scroll an animation glass behind a static breakup pattern for a textured, moving wash.
</Callout>

## Prism

A prism splits the beam along its axis, expanding one beam into several separated aerial shafts, each landing its own pool on the floor. Both `index` and `spin` always show on a prism row.

### Facet count

The facet count is inferred from the GDTF, in priority order: explicit `<Facet>` geometry, a leading number in the slot name (e.g. `4-Facet Circular Prism`), then a leading number in the wheel name, otherwise a fallback of `3`. A recognised count must be in the range `2–16` facets per prism.

<TypeTable
  type={{
    value: {
      description: 'Selects the slot, which slides into the shaft. Mid-travel a straight passthrough copy bleeds through alongside the facets.',
      type: '0–1',
      default: '0',
    },
    index: {
      description: 'Indexed rotation of the facet ring. Rotates the fan of beams to a fixed angle.',
      type: '0–1',
      default: '0',
    },
    spin: {
      description: 'Continuous rotation of the facet ring. 0.5 = stopped; off-centre spins the whole fan.',
      type: '0–1 (0.5 centre)',
      default: '0.5',
    },
  }}
/>

<Callout type="idea">
Insert the prism, set a slow `spin`, and the entire fan of beams sweeps around the room as one. Add `index` to lock the fan to a fixed angle instead.
</Callout>

### How the fan is built

Each facet produces one beam copy, deflected in the lens plane — the largest deflection is about `15°` off-axis (`tan ≈ 0.27`). Energy falls off sub-linearly across copies (`1/√n` per facet) so the fan stays punchy rather than dim. As the prism slides in or out, a passthrough copy of the main beam (weight `1 − insertion`) bleeds through, matching how a real prism looks while it travels.

### Stacking two prisms

A fixture with a second prism stacks them: each facet of the first multiplies against each facet of the second, so the beam count can grow quickly. The total is capped at `24` copies, keeping the highest-weight beams.

<Callout type="warn">
The total number of prism beam copies is capped at `24`. Stacking a high-facet prism on another can exceed that — copies beyond the cap are dropped, keeping the brightest.
</Callout>

## Frost

Frost slides in continuously rather than as a discrete slot: it ramps smoothly, softening the gobo edge and widening the beam toward a soft wash.

<TypeTable
  type={{
    value: {
      description: 'Diffusion amount. 0 = no frost, 1 = maximum. Softens the beam edge and spreads the cone wider.',
      type: '0–1',
      default: '0',
    },
  }}
/>

<Callout type="info">
Frost widens the cone (the half-angle scales by `1 + 0.4 × frost`) and pushes the edge toward a very soft falloff. If a fixture has multiple frost components, the strongest engaged one wins.
</Callout>

## Motion rates

Wheel motion is integrated once per frame, so transitions slew rather than snap.

| Motion | Rate |
| --- | --- |
| Gobo / wheel slot select | ~8 slots/s (about `0.1` s per slot) |
| Gobo image spin (full) | ~2.4 turns/s (≈876°/s) |
| Prism / frost insertion | full travel in ~`0.4` s |
| Shake oscillation | `1–9 Hz` |
| Animation scroll (full spin) | ~1.95 turns/s |
