## The default layout

A standard three-panel split with a data strip along the bottom. Every panel can be dragged, resized, hidden, or rearranged, and the arrangement is remembered per [workspace](/docs/interface/workspaces).

<Callout type="info">
The viewport is the only panel that can never be closed. Every other panel can be toggled from the **Window ▸ Panels** menu and reopens at its natural edge — Scene and Library on the left, Inspector on the right, the data panels along the bottom.
</Callout>

## The panels

### Scene

The [Scene outliner](/docs/scene/outliner) is a tree of everything in your project: the world, fixtures, objects (static geometry), screens, environments, and [pyro devices](/docs/fixtures/pyro). Rename items, toggle visibility, and build named [selection groups](/docs/scene/selection) here. Fixture rows carry an editable [sequence number](/docs/scene/outliner) (a channel-number-style id) that you can sort and renumber by. Type-filter chips along the top narrow the tree to **All / Fixtures / Objects / Screens / Pyro**; orthogonal state chips (Unpatched / Selected / Conflicts) further restrict fixtures.

### Library

The [Library](/docs/library) browses the fixture and content catalogue with search, sort, content-class chips (including **Imported** for GDTF you've brought in), and multi-select. It keeps persistent Recent and Favourites sections, and an **Add** button drops the selected templates into the scene. Single-click selects; `Cmd`/`Ctrl`+click multi-selects. (GDTF and MVR are imported from the **File** menu, not from this panel.)

### Viewport

The [viewport](/docs/navigation) renders the live 3D stage with DMX-driven lighting. It is the only panel with a header bar, which holds display-mode, exposure, grid, beam, snap, pivot, and transform-orientation controls.

### Inspector

The Inspector is a single property editor that adapts to the current selection — fixture optics, geometry transform, screen content, environment fog, pyro device, or the world (which also holds the render properties). It offers a property filter and a "show only modified" toggle, and it goes read-only while a still render is running.

## The menu bar

The menu bar runs across the top of the window. From left to right:

| Menu | What it covers |
| --- | --- |
| **File** | New Project, Open Project, Open Recent, Save (`Ctrl`+`S`), Save As, Import GDTF Fixture, Import / Export MVR Scene, Preferences. |
| **Edit** | Undo / Redo, Run Operator (`F3`), Adjust Last Operation (`F9`), Duplicate / Array, Delete Selected, and Select All (`A`) / Invert (`Ctrl`+`I`) / Deselect (`Alt`+`A`). |
| **View** | Camera (preset views), Frame Selection (`F`) / Frame All (`Shift`+`F`), Toggle Fullscreen (`F11`), and an Overlays submenu of toggles. |
| **Fixture** | Online Library and Edit Profile (the GDTF profile editor, enabled for a selected GDTF fixture). |
| **Render** | Render Image (`F12`), Render Animation (disabled), Cancel Render, Open Render View. |
| **Window** | Workspace, View Bookmarks, Performance, Report Log, Welcome, Panels, Reset Layout. |
| **Help** | Keyboard Shortcuts and About glowstone. |

For the full breakdown of every menu item, see [the Menu Bar reference](/docs/interface/menus).

<Callout type="warn">
**Render Animation** is visible in the Render menu but disabled — its tooltip reads "coming soon".
</Callout>

<Callout type="idea">
Saved projects use the native `.glow` format (magic `GLOW\0`), a single self-contained file holding the scene, preferences, the DMX patch and config, selection groups, cues and pyro devices. It's a self-describing MessagePack snapshot, so a show saved by any version of Glowstone opens in any other. See [Projects & the .glow Format](/docs/projects).
</Callout>

## The demo scene

A New Project starts with a small demo so you can see the renderer immediately: one **PAR Can 1** (a built-in PAR Can profile) up 4 m and tilted 45° at full intensity, inside one **Fog Box** haze volume resting on the floor. The renderer always draws a diffuse ground plane beneath it. It exercises the [beam engine](/docs/fixtures/beam-engine) and [volumetric beams](/docs/rendering/volumetrics).

## Reset and rearrange

If you drag the layout into a mess, **Window ▸ Reset Layout** restores the active workspace's saved arrangement. To switch between layouts, use the workspace tab strip or **Window ▸ Workspace**.
