## How shortcuts are resolved

Keymaps are organised into three contexts. When the same key is bound in more than one context, dispatch is **first-match-wins** in priority order:

<Steps>
<Step>
### Modal (highest priority)
Active only while a `G` / `R` / `S` transform is in progress. Only the axis-lock keys (`X` / `Y` / `Z`) and the hardcoded confirm/cancel keys fire. Global and Viewport bindings are suppressed, so `A` (Select all) or `S` (Quick-select) cannot fire mid-grab.
</Step>
<Step>
### Viewport
Active when the 3D Viewport is the focused panel. Masks Global bindings on the same key ŌĆö for example `S` is **Scale** here, not Quick-select.
</Step>
<Step>
### Global (lowest priority)
Always active unless a text field is editing. Used when no more specific context matches.
</Step>
</Steps>

<Callout type="info" title="Notes">
- `Cmd` on macOS = `Ctrl` on Windows/Linux. Shortcuts below are written as `Cmd`/`Ctrl`.
- Shortcuts do **not** fire while a text field, search box, or numeric input is actively editing.
- Some keys are context-dependent: `S` = Quick-select (Global) or Scale (Viewport); `Z` = Shading pie (Viewport) or axis lock (Modal).
</Callout>

<Callout type="idea">
Every binding here is rebindable, can be disabled, or extended with your own. See [Preferences & Theme](/docs/interface/preferences) for the Keymap editor; custom bindings persist to `keymap.json`.
</Callout>

Recall and save commands ship without a default key ŌĆö run them from the [Commands palette](/docs/interface/commands) / Window menu, or bind a key in the Keymap editor.

| Shortcut | Action |

</Callout>

## Nudge

Arrow keys move the selection in fixed world-space steps: a plain press = 0.1 m, with `Shift` = 1.0 m. Bursts within 600 ms coalesce into one undo step. Nudge only fires while the 3D Viewport is focused, something is selected, and no transform is in progress.

| Shortcut | Step | Action |
| --- | --- | --- |
| `Left Arrow` | 0.1 m | Nudge ŌłÆX |
| `Right Arrow` | 0.1 m | Nudge +X |
| `Up Arrow` | 0.1 m | Nudge ŌłÆZ |
| `Down Arrow` | 0.1 m | Nudge +Z |
| `PageUp` | 0.1 m | Nudge +height (Y up) |
| `PageDown` | 0.1 m | Nudge ŌłÆheight (Y down) |
| `Shift`+`Left Arrow` | 1.0 m | Nudge ŌłÆX |
| `Shift`+`Right Arrow` | 1.0 m | Nudge +X |
| `Shift`+`Up Arrow` | 1.0 m | Nudge ŌłÆZ |
| `Shift`+`Down Arrow` | 1.0 m | Nudge +Z |
| `Shift`+`PageUp` | 1.0 m | Nudge +height |
| `Shift`+`PageDown` | 1.0 m | Nudge ŌłÆheight |

## Rebinding shortcuts

<Steps>
<Step>
### Open the Keymap editor
Press `Cmd`/`Ctrl`+`,` to open [Preferences](/docs/interface/preferences) and switch to the Keymap section.
</Step>
<Step>
### Rebind, disable or add
Assign a command to a new trigger, disable a default, or add a brand-new binding in any context. Conflicts are detected and flagged before you save.
</Step>
<Step>
### Saved automatically
Every change persists immediately to `keymap.json`. A missing or corrupt file falls back to the shipped defaults.
</Step>
</Steps>

<Accordions type="single">
  <Accordion title="Why isn't my shortcut firing?">
    A text field is probably focused. While any search box, name input, or numeric field is editing, all shortcuts are suppressed so your keystrokes go into the field. Click elsewhere or press `Esc`, then try again.
  </Accordion>
  <Accordion title="Why does S do two different things?">
    Context. With the 3D Viewport focused, `S` enters Scale; otherwise it opens the Quick-select menu. The same context rule makes `Z` either the shading pie or, during a transform, the Z-axis lock.
  </Accordion>
  <Accordion title="Where do operator-only commands live?">
    Modal-only actions (`G`/`R`/`S`, axis lock, Camera view) and `kmi.*` keymap twins are not listed in the `F3` palette because they need live viewport state or would duplicate a canonical entry. Everything else is searchable in operator search.
  </Accordion>
</Accordions>
