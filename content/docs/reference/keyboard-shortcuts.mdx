## How shortcuts are resolved

Keymaps are organised into three contexts. When the same key is bound in more than one context, dispatch is **first-match-wins** in priority order:

<Steps>
<Step>
### Modal (highest priority)
Active only while a `G` / `R` / `S` transform is in progress. Only the axis-lock keys (`X` / `Y` / `Z`) and the hardcoded confirm/cancel keys fire. Global and Viewport bindings are suppressed, so `A` (Select all) or `S` (Quick-select) cannot fire mid-grab.
</Step>
<Step>
### Viewport
Active when the 3D Viewport is the focused panel. Masks Global bindings on the same key — for example `S` is **Scale** here, not Quick-select.
</Step>
<Step>
### Global (lowest priority)
Always active unless a text field is editing. Used when no more specific context matches.
</Step>
</Steps>

<Callout type="info" title="A few things to keep in mind">
- `Cmd` on macOS = `Ctrl` on Windows/Linux. Shortcuts below are written as `Cmd`/`Ctrl`.
- Shortcuts do **not** fire while a text field, search box, or numeric input is actively editing.
- Some keys are context-dependent: `S` = Quick-select (Global) or Scale (Viewport); `Z` = Shading pie (Viewport) or axis lock (Modal).
</Callout>

<Callout type="idea">
Every binding here is rebindable, can be disabled, or extended with your own. See [Preferences & Theme](/docs/interface/preferences) for the Keymap editor; custom bindings persist to `keymap.json`.
</Callout>

## Mouse — viewport navigation & selection

### Navigation

| Interaction | Action |
| --- | --- |
| `Drag` | Orbit — rotate the view around its centre |
| `Shift`+`Drag` | Pan — move the view horizontally/vertically |
| `Scroll` | Zoom — dolly in/out |
| `Shift`+`Right-click` | Place the [3D cursor](/docs/scene/transforming) at the ray-hit point |

<Callout type="info">
`Shift`+`Right-click` consumes the click: the context menu does **not** open and selection does **not** fire.
</Callout>

### Selection

| Interaction | Action |
| --- | --- |
| `Click` | Select — pick the object under the cursor |
| `Cmd`/`Ctrl`+`Click` | Toggle — add/remove from selection |
| `Shift`+`Click` | Range select — from the last pick to the current one |
| `Click`+`Drag` | Range box — drag a box to select multiple |

### Library & outliner

| Interaction | Action |
| --- | --- |
| `Drag` Library row | Drag a fixture from the [Library](/docs/library) panel into the viewport |
| Release `Drag` over viewport | Add the fixture at the cursor position |
| `Double-click` tree row | Rename — enter inline rename in the [Outliner](/docs/scene/outliner) |

## Numpad views & orbit

### Camera views (Viewport)

| Shortcut | Action |

| `Numpad 3` | Right view |
| `Cmd`/`Ctrl`+`Numpad 3` | Left view |
| `Numpad 7` | Top view |
| `Cmd`/`Ctrl`+`Numpad 7` | Bottom view |
| `Numpad 5` | Toggle orthographic / perspective |

### Stepped orbit (Viewport)

Each press orbits the camera by 15°.

| Shortcut | Action |
| --- | --- |
| `Numpad 8` | Orbit up (+15° pitch) |
| `Numpad 2` | Orbit down (−15° pitch) |

| `Numpad 6` | Orbit right (+15° yaw) |

## Pie menus

| Shortcut | Action |
| --- | --- |
| `` ` `` (backtick) | View pie — radial camera-view menu at the cursor |
| `Z` | Shading pie — display modes, grid and stats toggles |

The shading pie offers **Beauty**, **Unlit** and **Wireframe** display modes plus quick grid/stats toggles. See [Display Modes & Overlays](/docs/navigation/overlays).

<Callout type="info">
`Z` opens the shading pie in the Viewport, but during a `G`/`R`/`S` transform the same key locks the transform to the Z axis.
</Callout>

## Camera bookmarks

Camera (view) bookmarks store up to nine numbered camera poses (slots 1–9). They are distinct from [Cues](/docs/cues), which store fixture looks, and they persist to `bookmarks.json`.

| Shortcut | Action |
| --- | --- |
| (no default key) | Recall view bookmark N (eased transition) |
| (no default key) | Save current camera pose to the next free slot |

Recall and save commands ship without a default key — run them from the [Commands palette](/docs/interface/commands) / Window menu, or bind a key in the Keymap editor.

See [Camera Bookmarks](/docs/navigation/bookmarks) for details.

## Selection

| Shortcut | Action |
| --- | --- |
| `A` | Select all |
| `Alt`+`A` | Deselect all (none) |
| `Esc` | Deselect all (also cancels the active tool or modal op) |
| `Cmd`/`Ctrl`+`I` | Invert selection |
| `S` | Quick-select menu (Global context only) |
| `Shift`+`R` | Replace selected fixtures |

The Quick-select palette (`S` when the viewport is not focused) offers `A` all, `T` same type, `M` same maker, `I` invert, `N` none. See [Selecting Objects](/docs/scene/selection).

<Callout type="warn">
`S` only opens Quick-select when the viewport is **not** focused. With the viewport focused, `S` enters Scale instead.
</Callout>

## Transform

Modal transforms run live in the Viewport. Type a number during the grab to set an exact amount (distance for Move, degrees for Rotate, factor for Scale).

### Enter a transform (Viewport)

| Shortcut | Action |
| --- | --- |
| `G` | Move (grab) |
| `R` | Rotate |
| `S` | Scale |

### Axis lock (Modal — only during a transform)

| Shortcut | Action |
| --- | --- |
| `X` | Lock to X axis |
| `Y` | Lock to Y axis |
| `Z` | Lock to Z axis (height) |

### Confirm / cancel (Modal)

| Shortcut | Action |
| --- | --- |
| `Enter` | Confirm the transform |
| `Space` | Confirm (alternative) |
| `Esc` | Cancel and revert |
| `Right-click` | Cancel and revert |

<Callout type="info">
`Space` confirms during a transform; everywhere else it opens the operator search. The dispatcher suppresses operator search while a transform is active so `Space` is free to confirm.
</Callout>

See [Moving, Rotating & Scaling](/docs/scene/transforming).

## Nudge

Arrow keys move the selection in fixed world-space steps: a plain press = 0.1 m, with `Shift` = 1.0 m. Bursts within 600 ms coalesce into one undo step. Nudge only fires while the 3D Viewport is focused, something is selected, and no transform is in progress.

| Shortcut | Step | Action |
| --- | --- | --- |
| `Left Arrow` | 0.1 m | Nudge −X |
| `Right Arrow` | 0.1 m | Nudge +X |
| `Up Arrow` | 0.1 m | Nudge −Z |
| `Down Arrow` | 0.1 m | Nudge +Z |
| `PageUp` | 0.1 m | Nudge +height (Y up) |
| `PageDown` | 0.1 m | Nudge −height (Y down) |
| `Shift`+`Left Arrow` | 1.0 m | Nudge −X |
| `Shift`+`Right Arrow` | 1.0 m | Nudge +X |
| `Shift`+`Up Arrow` | 1.0 m | Nudge −Z |
| `Shift`+`Down Arrow` | 1.0 m | Nudge +Z |
| `Shift`+`PageUp` | 1.0 m | Nudge +height |
| `Shift`+`PageDown` | 1.0 m | Nudge −height |

## Rebinding shortcuts

<Steps>
<Step>
### Open the Keymap editor
Press `Cmd`/`Ctrl`+`,` to open [Preferences](/docs/interface/preferences) and switch to the Keymap section.
</Step>
<Step>
### Rebind, disable or add
Assign a command to a new trigger, disable a default, or add a brand-new binding in any context. Conflicts are detected and flagged before you save.
</Step>
<Step>
### Saved automatically
Every change persists immediately to `keymap.json`. A missing or corrupt file falls back to the shipped defaults.
</Step>
</Steps>

<Accordions type="single">
  <Accordion title="Why isn't my shortcut firing?">
    A text field is probably focused. While any search box, name input, or numeric field is editing, all shortcuts are suppressed so your keystrokes go into the field. Click elsewhere or press `Esc`, then try again.
  </Accordion>
  <Accordion title="Why does S do two different things?">
    Context. With the 3D Viewport focused, `S` enters Scale; otherwise it opens the Quick-select menu. The same context rule makes `Z` either the shading pie or, during a transform, the Z-axis lock.
  </Accordion>
  <Accordion title="Where do operator-only commands live?">
    Modal-only actions (`G`/`R`/`S`, axis lock, Camera view) and `kmi.*` keymap twins are not listed in the `F3` palette because they need live viewport state or would duplicate a canonical entry. Everything else is searchable in operator search.
  </Accordion>
</Accordions>
