An LED screen displays content from **one source at a time**. Choosing a source in the Inspector tells Glowstone what to push to the wall's texture, whether that is a procedural pattern, a still image, or a live network stream. Switching sources drops any live frame from the previous source.

## The six sources at a glance

| Source | What it is | Live? | Set up where |
| --- | --- | --- | --- |
| Test Pattern | Procedural Grid / Colour Bars / Gradient | No | Inspector → Content |
| Solid Colour | Single linear-RGB fill | No | Inspector → Content (colour picker) |
| Image | Still image bundled in the project | No | Inspector → Content (`Choose image…`) |
| NDI | Live network video receive | Yes | [NDI Video](/docs/media/ndi) |
| CITP | Live media-server stream | Yes | [Media Servers](/docs/media/media-servers) |
| Pixel-map DMX | Low-res RGB control grid | Yes | [Pixel-Map DMX](/docs/media/pixel-mapping) |

## Switch a screen's source

<Steps>
<Step>
### Select the screen
Click the LED screen in the viewport or in the [Outliner](/docs/scene/outliner). Its properties appear in the Inspector.
</Step>
<Step>
### Open the Content section
In the Inspector, expand the **Content** section. The first control is the **Source** dropdown.
</Step>
<Step>
### Pick a source
Choose one of the six sources. The controls below the dropdown change to match the source you selected.
</Step>
<Step>
### Configure the source
For procedural sources, set the pattern or colour inline. For live sources, pick a discovered source from the picker or type a name. See the dedicated pages for [NDI](/docs/media/ndi), [CITP](/docs/media/media-servers) and [Pixel-Map DMX](/docs/media/pixel-mapping).
</Step>
</Steps>

<Callout type="warn">
Switching the source clears any live frame held from the previous source. The wall goes to the new source's content immediately; a live stream takes a moment to reconnect and deliver its first frame.
</Callout>

## Procedural sources

Generated by Glowstone directly, so they carry no streamed frame data.

### Test Pattern

A procedural pattern drawn in the shader, for alignment and commissioning before real content exists.

<TypeTable
  type={{
    Grid: { description: 'Per-pixel alignment grid (crosshatch plus cabinet ticks) on a dim blue field.', type: 'Pattern', default: 'Selected' },
    'Colour Bars': { description: 'SMPTE-style: 7 vertical bars (white, yellow, cyan, green, magenta, red, blue). Use for per-channel verification.', type: 'Pattern', default: '' },
    Gradient: { description: 'Smooth horizontal hue sweep with a luma ramp. Use for geometry and colour-space checks.', type: 'Pattern', default: '' },
  }}
/>

<Callout type="idea">
After setting cabinet size and pixels-per-cabinet to match a spec sheet, run **Colour Bars** to verify per-channel response, then **Gradient** to check the colour space, then zoom in to inspect the pixel shape (SMD round, SMD square, or discrete RGB).
</Callout>

### Solid Colour

A single fill from an RGB colour picker, interpreted in **linear RGB**. The default fill is a blue at `[0.1, 0.4, 0.9]`. Use it as a wash or backdrop.

### Image

A still image chosen with the **`Choose image…`** button. Glowstone shows the loaded filename, or `no image` when none is set.

<TypeTable
  type={{
    Formats: { description: 'Supported still formats decoded on load.', type: 'PNG, JPG/JPEG, BMP, GIF, WebP, TGA, EXR, HDR', default: '—' },
    Storage: { description: 'The image bytes are bundled into the .glow project file, so the screen restores without the original file.', type: 'In-project', default: '—' },
    Refresh: { description: 'Not a live source. The image is decoded once on load and does not update.', type: 'Static', default: '—' },
  }}
/>

<Callout type="info">
Because the image is stored in the project, the wall keeps its content even if the original file moves or is deleted. See [Projects & the .glow Format](/docs/projects) and [Supported File Formats](/docs/projects/file-formats).
</Callout>

## Live sources

Three sources deliver frames over the network or from DMX input. Each has its own setup page.

<Tabs items={['NDI', 'CITP', 'Pixel-map DMX']}>
  <Tab value="NDI">
    Receives live video from an NDI source on the LAN. Pick a discovered source from the picker or type its source name (e.g. `"HOST (Output 1)"`). Requires an NDI-enabled build (`--features ndi`) and the installed NDI runtime; otherwise the picker reports that the NDI runtime is not available. Full details on the [NDI Video](/docs/media/ndi) page.
  </Tab>
  <Tab value="CITP">
    Streams from a CITP/MSEX media server auto-discovered on the LAN. Identify the source as `"Server | layer"` (e.g. `"Resolume Arena | 1"`); a bare name defaults to layer `0`. Full details on the [Media Servers](/docs/media/media-servers) page.
  </Tab>
  <Tab value="Pixel-map DMX">
    Drives a low-resolution RGB cell grid from incoming Art-Net / sACN. Set the grid size (cols × rows) and the universe / start address; each cell reads 3 channels (RGB). It is not real video and is always stretched to fill the wall. Full details on the [Pixel-Map DMX](/docs/media/pixel-mapping) page.
  </Tab>
</Tabs>

<Callout type="info" title="DMX is input-only">
Glowstone visualizes DMX it receives; it never outputs or transmits DMX. Pixel-map DMX reads incoming Art-Net / sACN to colour the grid. See [DMX Overview](/docs/dmx) and [Connectivity](/docs/dmx/connectivity).
</Callout>

## How the wall lights the scene

A screen does not light the stage per pixel. Instead, Glowstone computes an average **summary colour** from the content and applies it as a single, cheap area-light contribution, which is what feeds volumetric haze and fog for beams in front of the wall.

<TypeTable
  type={{
    'Light emit': { description: 'Per-screen multiplier on how much the wall illuminates the scene and contributes to haze. 0 = lights nothing (purely visual display).', type: '0–4.0', default: '1.0' },
  }}
/>

<Callout type="info">
Because lighting comes from an averaged summary colour, a busy video clip and a solid fill of the same average colour wash the haze the same way. Tune the wall's haze contribution with `Light emit`. For how beams pick up that light, see [Volumetric Beams & Haze](/docs/rendering/volumetrics) and [Environments: Fog & Haze](/docs/scene/environment).
</Callout>
