## Opening the editor

<Steps>
<Step>
### Select the fixture
Click the fixture in the 3D viewport or the [Scene Outliner](/docs/scene/outliner).
</Step>
<Step>
### Open the profile
With a GDTF fixture selected, choose **Fixture ▸ Edit Profile…**, or click the **Profile…** button in the fixture's [Inspector](/docs/interface/panels) header.
</Step>
</Steps>

<Callout type="info" title="Read-only by design">
Almost everything in the editor comes straight from the GDTF file and is read-only. Only a fixture's **instance name**, **Fixture ID** (MVR fixtures only), **DMX mode** and **Move speed** are editable per placement — the GDTF profile itself is never modified.
</Callout>

The editor is organised into seven tabs.

## Identity

The Identity tab shows who made the fixture and how this placement is labelled. The descriptive fields are read-only; the instance-specific fields can be edited. A fixture's GDTF **Description**, when present, is shown in a separate section below the grid.

<Callout type="idea">
Set a clear **Instance name** for every fixture. It is the label you will see in the [Scene Outliner](/docs/scene/outliner), in [patching](/docs/dmx/patching) and on the [DMX Monitor](/docs/dmx/dmx-monitor).
</Callout>

### Modelled movement

Below the optical chain a section marked **MODELED — NOT IN GDTF** carries the synthesized movement values, each with a *modeled* badge. The GDTF carries no real degrees-per-second figure, so Glowstone derives one so moving fixtures feel realistic.

<TypeTable
  type={{
    'Move speed': { description: 'Motor-speed control for pan/tilt. Editable per instance: 0 = fastest (tracking), 1 = slowest.', type: 'slider 0–1', default: 'editable' },
    'pan / tilt': { description: 'Resulting maximum pan / tilt slew rates, derived from Move speed and the fixture class.', type: '°/s (read-only)' },
  }}
/>

### Emitter selector

A column on the left, headed **EMITTERS**, lists the emitters (light sources) in the current mode.

- For multi-emitter fixtures, each emitter is listed by index and name. Click one to see its individual beam optics.
- Single-source fixtures show **(single source)**.

<Callout type="info">
The number of emitters depends on the selected DMX mode. Switching modes on the [DMX Modes](#dmx-modes) tab can change which emitters appear here.
</Callout>

## DMX modes

The DMX Modes tab lists every DMX mode the fixture type supports. Each row shows the mode name, its channel count (footprint) and its emitter count.

<TypeTable
  type={{
    'Mode name': { description: 'The mode label.', type: 'e.g. Standard, Extended' },
    'Channel count': { description: 'Number of DMX slots the mode occupies (footprint).', type: 'e.g. 16 ch' },
    'Emitter count': { description: 'Number of light sources active in this mode.', type: 'e.g. 4 emitters' },
  }}
/>

**Click a mode to switch the fixture to it.** This is a per-instance setting: it reconfigures which channels are available and updates the geometry, emitter and wheel assignments. The fixture stays placed in the scene — only its control layout changes.

<Callout type="warn">
Changing the mode changes the channel footprint. If the fixture is already patched, review its address afterwards so it does not overlap a neighbour. See [Patching Fixtures](/docs/dmx/patching).
</Callout>

For a deeper look at how modes and channels relate, see [DMX Modes & Channels](/docs/library/dmx-modes).

## Channels

The Channels tab is a table of every DMX channel in the current mode, with the mode name and footprint shown above it. Rows are **resolved** — for multi-emitter or instanced geometries, each instance gets its own DMX address.

| Column | Meaning |
| --- | --- |
| `Addr` | DMX slot number within the fixture footprint (1-based). Shows `—` for a virtual channel with no footprint. |
| `Geometry` | The geometric component the channel controls — the reference-instance name when instanced, e.g. `Pan`, `Tilt`, `Lens2`. |
| `Attribute` | The control attribute name, e.g. `Pan`, `Tilt`, `Dimmer`, `Color1`. |
| `Function` | The primary channel-function name, e.g. `Pan`, `Focus`, `Dimmer`. |

<Callout type="info">
A single logical channel may occupy more than one slot. 16-bit attributes such as `Pan` and `Tilt` use a coarse and a fine slot, so the channel count can exceed the number of distinct attributes.
</Callout>

To map attributes to live beam behaviour, see the [Beam Parameter Reference](/docs/fixtures/parameters).

## Wheels

The Wheels tab previews every optical wheel defined in the GDTF — colour, gobo, prism, animation and frost — with a thumbnail for each slot. Fixtures with no wheels show **No wheels.**

Each wheel header shows its name and slot count, e.g. 'Gobo1 · 8 slots'. The cursor hovers a gobo slot showing its name tooltip."
/>

Each wheel header shows its **name** (e.g. `Colour1`, `Gobo1`, `Prism1`) and **slot count** (e.g. `8 slots`). For every slot:

- If slot media is available (a gobo image or animation frame), the PNG thumbnail is shown.
- Otherwise a colour swatch is drawn from the GDTF CIE xyY colour data, converted to RGB.
- Hover a slot to see its name (e.g. `Open`, `Red`, `CTO`).

For how these wheels drive the rendered beam, see [Gobos, Animation, Prism & Frost](/docs/fixtures/gobos-prism).

## Defaults

The Defaults tab shows the DMX default value for each channel in the current mode, taken from the GDTF `InitialFunction`. These are the values the fixture holds before any console data arrives.

<TypeTable
  type={{
    Attribute: { description: 'The channel attribute.', type: 'read-only' },
    'Default value': { description: 'The power-on default, normalized to a 0–1 range (shown to 3 decimals).', type: '0–1 (read-only)' },
  }}
/>

<Callout type="info">
Glowstone only visualises DMX from a console — it never outputs DMX. These defaults define each fixture's starting look until live values arrive over [Art-Net or sACN](/docs/dmx/connectivity).
</Callout>

## Frequently asked

<Accordions type="single">
  <Accordion title="Can I edit a fixture's beam angle or colour temperature?">
    The Optics values shown come from the GDTF and are read-only. The editable per-instance settings are the instance name, Fixture ID (MVR fixtures only), DMX mode and the modelled Move speed.
  </Accordion>
  <Accordion title="Why does the channel count change when I switch modes?">
    Each DMX mode is a distinct control configuration with its own channel footprint and emitter set. Switching modes reconfigures the channel map, so the totals on the Channels and Defaults tabs change to match.
  </Accordion>
  <Accordion title="A wheel slot shows a flat colour instead of a gobo. Why?">
    The GDTF for that slot carries no media image, so Glowstone draws a swatch from the slot's CIE colour data instead. Slots with embedded PNG media show the image.
  </Accordion>
</Accordions>
