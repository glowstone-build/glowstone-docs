Every GDTF fixture ships with one or more **DMX modes**. Each mode is a complete control configuration: how many DMX slots the fixture occupies and which attributes you can control. Switching mode changes the footprint and the available channels.

## The GDTF control model

GDTF describes a fixture's controls as a small hierarchy. Understanding the layers makes the Profile editor's tabs easier to read.

<Cards>
  <Card icon={<Layers />} title="DMX mode">
    A control configuration with a fixed channel count (footprint), a set of emitters and an optical wheel chain.
  </Card>
  <Card icon={<SlidersHorizontal />} title="Channel">
    One attribute (Pan, Dimmer, Color1…) occupying one or more DMX slots in the mode.
  </Card>
  <Card icon={<Hash />} title="Channel function">
    A DMX sub-range within a channel, mapped to a physical range or a wheel slot.
  </Card>
</Cards>

A fixture's mode lives at the top. Each mode contains channels; each channel contains one or more channel functions. The Profile editor exposes all of this — see [The Profile Editor](/docs/library/profile-editor) for the full tab-by-tab tour.

<Callout type="info">
Glowstone is input-only for DMX. It reads incoming DMX from your console and visualises it against each fixture's mode. It never outputs or transmits DMX.
</Callout>

## DMX modes

A mode is a named control layout. A moving head might offer `Standard` (16 channels), `Extended` (32 channels) and a `Low Resolution` variant. The mode determines:

- **Footprint** — total DMX slots the fixture consumes.
- **Channels** — which attributes are addressable, and at what resolution.
- **Emitters** — how many light sources the fixture drives in this mode.
- **Optical chain** — the colour, gobo, prism, animation and frost wheels exposed.

### Switching mode

You change a fixture's mode per instance in the Profile editor. The fixture stays placed in the scene; only its control layout changes.

<Steps>
<Step>
### Open the Profile editor
Select the GDTF fixture, then click **Profile…** in the inspector — or use the menu bar **Fixture → Edit Profile…**.
</Step>
<Step>
### Go to the DMX Modes tab
The tab lists every mode for this fixture type. Each row reads `Name · N ch · N emitters`.
</Step>
<Step>
### Click a mode
The fixture switches to that mode for this instance, and its footprint, channels and emitters reconfigure to match.
</Step>
</Steps>

<Callout type="warn">
Switching mode changes the footprint. After changing mode, re-check the fixture's patch — the address range it occupies may have grown or shrunk. See [Patching Fixtures](/docs/dmx/patching).
</Callout>

## Channels

Each channel binds one **attribute** to one or more DMX slots. Pan, Tilt, Dimmer and Color1 are all channels. The Channels tab shows them as a resolved table: address, geometry, attribute and function.

### 8-bit and 16-bit channels

A channel's resolution is how many slots it uses: an 8-bit channel occupies one slot, a 16-bit channel two (coarse + fine).

<TypeTable
  type={{
    attribute: { description: 'Primary attribute name, e.g. "Pan", "Tilt", "Dimmer", "Color1".', type: 'text' },
    function: { description: "Human-readable name of the channel's primary range, e.g. \"Pan\", \"Focus\", \"Dimmer\".", type: 'text' },
    geometry: { description: 'The geometric component this channel drives, e.g. "Pan Axis", "Base Fixture", "Zone2".', type: 'text' },
    offsets: { description: 'The 1-based DMX slot indices this channel occupies. Two slots for a 16-bit channel.', type: 'slot(s)' },
    resolution: { description: 'Byte resolution. 1 = 8-bit, 2 = 16-bit. Equals the number of offsets.', type: '1 | 2 …', default: '1' },
    default: { description: 'Default value the fixture rests at (from the GDTF InitialFunction), normalised 0–1.', type: '0–1' },
    dmx_break: { description: 'DMX break number for instancing. None means "Overwrite".', type: 'number | None' },
  }}
/>

<Callout type="idea">
A 16-bit Pan or Tilt channel gives 65,536 steps instead of 256 — the difference between visibly stepped moves and smooth slews when a beam crawls across the stage.
</Callout>

## Channel functions

A single channel rarely does just one thing. **Channel functions** carve the channel's DMX range into sub-ranges, each mapped to a physical range or a wheel slot. On a colour wheel channel, one band of DMX values selects red, another selects blue, and a high band might spin the wheel.

<TypeTable
  type={{
    name: { description: 'Function name, human-readable, e.g. "Pan", "Open", "Red".', type: 'text' },
    attribute: { description: 'Attribute this function belongs to, e.g. "Pan".', type: 'text' },
    dmx_from: { description: 'DMX value where this sub-range starts, normalised 0–1 over the full channel.', type: '0–1' },
    physical_from: { description: 'Physical range start, e.g. -270 for pan degrees.', type: 'number' },
    physical_to: { description: 'Physical range end, e.g. 270 for pan degrees.', type: 'number' },
    wheel: { description: 'Linked wheel name for wheel-based functions, e.g. "Colour1".', type: 'text | none' },
  }}
/>

For wheel channels, each function can carry a list of **channel sets** — the exact `DMX → wheel-slot` mappings from the GDTF. These drive the slot names shown in the inspector and the swatches/gobo thumbnails on the Wheels tab.

<Accordions type="single">
  <Accordion title="What is a channel set?">
    A channel set is one GDTF `<ChannelSet>` row: a DMX start value (normalised 0–1) paired with a wheel slot index (1-based; 0 means no slot link). It is how a discrete DMX value such as "value 32 = Gobo 1" is recorded.
  </Accordion>
  <Accordion title="Why does my channel show physical ranges in degrees?">
    Continuous attributes like Pan and Tilt map their DMX range to a physical range — for example -270° to +270° of pan travel. The channel function's physical_from / physical_to define that mapping.
  </Accordion>
  <Accordion title="Where do gobo and prism controls come from?">
    They come from the mode's optical chain rather than from a plain physical range. Wheel controls split into Value (slot selection or insertion %), Index (indexed rotation of the chosen slot) and Spin (continuous rotation, where 0.5 = stop). See Gobos, Animation, Prism & Frost.
  </Accordion>
</Accordions>

## Emitters and instanced cells

A mode lists its **emitters** — its light sources. Single-source fixtures show "(single source)". Multi-emitter fixtures (LED bars, blinders, flower effects) drive several beams.

When a fixture uses `GeometryReference` instancing — for example a multi-zone LED bar — each instanced cell gets its **own DMX address**. The Channels tab shows these as resolved rows: the same attribute repeats once per cell, each at a distinct slot.

<Callout type="info">
This is why a "4-cell" bar in a multi-zone mode consumes four times the per-cell channel count. Each zone is independently addressable, so each zone's Dimmer, Red, Green and Blue land at their own offsets.
</Callout>

## Defaults

The Defaults tab lists each channel's resting value, taken from the GDTF `InitialFunction` and normalised to a 0–1 range. These are the values the fixture holds before a console takes control — useful for confirming that, say, a shutter sits open and a colour wheel sits on `Open` at rest.
